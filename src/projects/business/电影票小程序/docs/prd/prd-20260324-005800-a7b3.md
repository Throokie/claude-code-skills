## 2. 产品概述

### 2.1 产品背景

将现有的电影排期展示系统扩展为完整的电影票销售小程序。

**核心理念**: 这是一个"极度偏心、高度控场、懂人性的智能过滤器"，而非简单的买票软件。

**解决的问题**:
- 用户购票体验差，影院信息不透明
- 外部调票出票时间长，用户预期不明确
- 管理员多渠道出票管理混乱
- 订单处理效率低，沟通成本高

### 2.2 产品目标

- **目标1**: 影院推荐准确率 > 90%（按利润率、成功率排序）
- **目标2**: 订单支付成功率 > 95%
- **目标3**: 管理员出票处理时间 < 5分钟
- **目标4**: 用户满意度 > 4.5/5

### 2.3 目标用户

| 用户类型 | 特征 | 主要需求 |
|---------|------|---------|
| 普通观影用户 | 想要购买电影票的消费者，追求性价比 | 快速找到优惠影院、便捷下单支付、实时了解订单状态 |
| 管理员/店长 | 处理订单、出票的运营人员 | 高效处理订单、多渠道出票选择、一键复制话术 |
| 影院经理 | 提供票务资源的合作方 | 便捷接单、快速沟通 |

### 3.2 详细功能描述

#### F-001: 影院推荐排序

**用户故事**: 作为普通观影用户，我想要看到按推荐排序的影院列表，以便快速找到高性价比的观影选择。

**验收标准**:
- [ ] 影院按推荐权重排序（利润率、成功率、出票速度综合计算）
- [ ] 推荐影院显示在前，外部调票影院后置
- [ ] 支持手动刷新列表

**排序算法**:
```
推荐分数 = (利润率 * 0.4) + (成功率 * 0.35) + (出票速度 * 0.25)
```

**边界情况**:
- 新影院无历史数据: 使用默认中位值计算
- 数据异常: 降级为按影院名称排序

---

#### F-002: 标签系统

**用户故事**: 作为普通观影用户，我想要通过标签快速识别影院类型，以便做出更好的购票决策。

**验收标准**:
- [ ] 显示4种标签：🔥店长直供、💰今日特价、⚡秒出票、[外部调票]
- [ ] 标签根据影院属性自动计算并显示
- [ ] 标签优先级：店长直供 > 今日特价 > 秒出票 > 外部调票

**标签规则**:
| 标签 | 条件 |
|------|------|
| 🔥店长直供 | 有稳定合作渠道，出票成功率 > 95% |
| 💰今日特价 | 利润率 > 20% |
| ⚡秒出票 | 平均出票时间 < 10分钟 |
| [外部调票] | 无稳定渠道，需外部调票 |

#### F-004: 下单与支付

**用户故事**: 作为普通观影用户，我想要选择座位偏好并完成支付，以便快速购买电影票。

**验收标准**:
- [ ] 支持选择座位偏好（靠前排/靠后排/中间位置）
- [ ] 订单30分钟支付超时自动取消
- [ ] 集成微信支付v3 API
- [ ] 支付成功后跳转订单详情

**订单字段**:
- 影院ID、影院名称
- 电影名称、场次时间、影厅
- 座位数量、座位偏好
- 单价、总价
- 备注信息

**边界情况**:
- 支付超时: 订单自动取消，释放库存
- 支付失败: 保留订单15分钟，允许重新支付

#### F-005: 订单状态跟踪

**用户故事**: 作为普通观影用户，我想要实时查看订单处理状态，以便了解出票进度。

**验收标准**:
- [ ] 状态流转：pending → paid → accepted → ticketed → completed
- [ ] 每个状态有清晰的视觉提示
- [ ] 待接单状态显示沙漏动画
- [ ] 已接单状态显示进度条
- [ ] 状态变更实时推送通知

**状态说明**:
| 状态 | 说明 | 视觉 |
|------|------|------|
| pending | 待支付 | 倒计时显示 |
| paid | 已支付，待接单 | 沙漏动画 |
| accepted | 已接单 | 进度条 |
| ticketed | 已出票 | 座位号+暗号 |
| completed | 已完成 | 观影结束 |
| refunded | 已退款 | 退款金额 |
| cancelled | 已取消 | - |

**渠道选择**:
| 渠道 | 使用场景 |
|------|----------|
| 影院经理 | 一键复制话术，微信沟通 |
| API渠道 | 麻花/聚视等API下单 |
| 自己找券 | 闲鱼/淘宝/羊毛群 |

#### F-007: 暗号系统

**用户故事**: 作为普通观影用户，我想要通过暗号快速取票，以便提升取票体验。

**验收标准**:
- [ ] 支持两种暗号：祝福语（财源广进、平安喜乐等）、数字（888、666等）
- [ ] 待出票状态：暗号隐藏显示（****）
- [ ] 已出票状态：暗号高亮显示
- [ ] 开场后2小时：暗号降级（灰色/折叠）

**暗号生命周期**:
```
待出票 → 隐藏（****）
    ↓
已出票 → 高亮显示
    ↓
开场后2小时 → 灰色/折叠
```

#### F-008: 低优先级影院警告

**用户故事**: 作为普通观影用户，我想要在选择外部调票影院时收到警告，以便有合理的时间预期。

**验收标准**:
- [ ] 点击外部调票影院时弹出警告
- [ ] 警告内容："该影院为外部调票，出票时长约15-30分钟"
- [ ] 提供两个选项：返回推荐影院 / 继续下单
- [ ] 用户选择"继续"后记录决策

**设计意图**: 通过前置警告降低用户预期，减少催单和投诉。

---

#### F-009: 拒绝订单+补偿券

**用户故事**: 作为管理员，我想要在无法出票时拒绝订单并自动退款，以便维护用户关系。

**验收标准**:
- [ ] 支持填写退款金额、补偿券金额、退款原因
- [ ] 确认后立即自动退款
- [ ] 用户收到退款通知+补偿券到账通知
- [ ] 补偿券计入用户 coupon_balance

**补偿策略**:
- 退款金额 = 实际支付金额
- 补偿券 = 5-20元（根据情况）
- 补偿券可用于下次购票抵扣

---

#### F-010: 运营状态管理

**用户故事**: 作为管理员，我想要快速切换运营状态，以便应对不同业务场景。

**验收标准**:
- [ ] 支持三种状态：正常接单、人工超载（爆单）、打烊休息
- [ ] 状态切换立即生效
- [ ] 爆单模式显示公告信息
- [ ] 打烊状态禁止新订单

**状态说明**:
| 状态 | 行为 |
|------|------|
| 正常接单 | 正常接收和处理订单 |
| 人工超载 | 显示爆单公告，可配置是否暂停接单 |
| 打烊休息 | 显示休息提示，禁止新订单 |

### 5.3 数据模型

class Cinema(SQLModel, table=True):

## 6. UI/UX 设计

### 6.1 设计原则
- **简洁直观**: 用户3秒内理解如何操作
- **视觉层次**: 重要信息突出，次要信息弱化
- **及时反馈**: 操作后即时响应，状态清晰可见
- **容错设计**: 错误操作可撤销，边界情况友好提示

### 6.2 页面流程

#### 用户端流程
```
首页（排期概览）
    ↓
影院详情页
    ↓
场次选择页
    ↓
下单页（座位偏好、备注）
    ↓
支付页（微信支付）
    ↓
订单详情页（状态跟踪）
    ↓
[已出票] 显示座位号+暗号
```

#### 管理后台流程
```
登录页
    ↓
订单列表页（新订单高亮）
    ↓
订单详情页
    ↓
[接单] → 渠道选择 → 录入信息 → 完成
[拒绝] → 填写原因 → 确认退款
```

### 6.3 关键页面说明

#### 首页
- 顶部：搜索栏、个人中心入口
- 中部：公告栏（运营状态）
- 主体：影院卡片列表（按优先级排序）
- 卡片内容：影院名称、标签、起价、距离

#### 订单详情页
- 头部：订单号、状态标签
- 影院信息：名称、地址、场次
- 座位信息：数量、偏好、具体座位（出票后显示）
- 支付信息：金额、支付方式
- 状态进度条：可视化状态流转
- 操作按钮：根据状态显示不同操作

#### 管理后台-订单列表
- 筛选栏：状态、时间、影院
- 订单卡片：订单号、影院、金额、状态、时间
- 新订单提示：声音+视觉提示
- 快捷操作：接单/拒绝按钮
