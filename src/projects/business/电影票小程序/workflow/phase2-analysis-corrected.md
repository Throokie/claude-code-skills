## 2. 用户痛点分析

### 用户端痛点 (严重程度: 高)

| 痛点 | 严重程度 | 当前问题 | 期望结果 |
|------|----------|----------|----------|
| 购票体验差 | 高 | 传统购票流程繁琐，影院信息不透明 | 智能推荐，一键下单 |
| 出票时间长 | 高 | 外部调票等待时间长，用户体验差 | 分级提示，合理预期 |
| 取票流程复杂 | 中 | 需要排队取票，容易丢失电子票 | 暗号系统，快速取票 |
| 订单状态不透明 | 中 | 不知道订单处理进度 | 实时状态更新，推送通知 |
| 退款流程慢 | 中 | 订单拒绝后退款周期长 | 即时退款+补偿券 |

### 管理员端痛点 (严重程度: 高)

| 痛点 | 严重程度 | 当前问题 | 期望结果 |
|------|----------|----------|----------|
| 多渠道管理难 | 高 | 影院经理、API、自己找券多渠道切换 | 统一界面，快速选择 |
| 订单处理效率低 | 高 | 手动录入信息，沟通成本高 | 一键复制话术，快速出票 |
| 运营状态切换麻烦 | 中 | 爆单时无法快速暂停接单 | 一键切换运营状态 |

## 3. 关键使用场景

### 场景A: 主流观影流程 (80%用户)

**触发条件**: 用户想要购买电影票
**用户期望**: 快速找到心仪影院并下单
**边界情况**:
- 用户未在30分钟内支付 → 订单自动取消
- 管理员拒绝订单 → 自动退款+补偿券
- 开场前2小时仍未出票 → 紧急提醒管理员

```
用户打开小程序
    ↓
首页显示今日排期概览（按推荐排序）
    ↓
点击推荐影院 → 查看影院详情和排期
    ↓
选择电影场次 → 进入下单页
    ↓
选择座位偏好、票数、备注
    ↓
确认支付 → 微信支付 → 支付成功
    ↓
跳转订单详情 → 查看待接单状态
    ↓
管理员接单 → 出票 → 推送通知
    ↓
用户查看取票暗号 → 到店取票观影
```

### 场景B: 低优先级影院选择 (20%用户)

**触发条件**: 用户点击外部调票影院
**用户期望**: 了解出票时间预期
**边界情况**:
- 用户选择返回推荐影院
- 用户选择继续下单（有心理预期）

```
用户点击外部调票影院
    ↓
弹窗警告："该影院为外部调票，出票时长约15-30分钟"
    ↓
用户选择：返回推荐影院 / 继续下单
    ↓
如继续 → 正常下单流程（但有心理预期）
```

**设计意图**: 通过前置警告降低用户预期，减少催单和投诉。

### 场景C: 管理员后台操作流程

**触发条件**: 收到新订单通知
**用户期望**: 快速处理订单并出票
**边界情况**:
- 无法出票 → 拒绝并退款
- 多渠道选择 → 影院经理/API/自己找券

```
收到新订单通知（飞书/微信推送）
    ↓
打开管理后台 → 订单列表页
    ↓
点击订单卡片 → 查看订单详情
    ↓
点击【接单】→ 进入渠道选择
    ↓
选择出票渠道（影院经理/API/自己找券）
    ↓
出票成功 → 录入座位号+选择暗号
    ↓
系统推送通知用户"已出票"
```

## 4. 竞品洞察

| 竞品 | 如何解决 | 我们的差异化 |
|------|----------|--------------|
| **猫眼电影** | 标准购票流程，即时出票 | 我们提供更低价格的外部调票渠道 |
| **淘票票** | 优惠券+积分体系 | 我们的暗号系统更有趣 |
| **微信小程序购票** | 便捷入口 | 我们专注服务小影院，更灵活 |
| **闲鱼电影票** | 个人卖家出票 | 我们提供系统化管理，更可靠 |

**差异化优势**:
1. **智能过滤器** - 根据利润率和成功率推荐影院
2. **暗号系统** - 增加趣味性和仪式感
3. **补偿券机制** - 提升用户满意度
4. **多渠道出票** - 灵活应对各种情况

## 5. 技术风险与建议

### 风险1: 微信支付集成
- **风险**: 微信支付v3 API集成复杂，证书管理麻烦
- **建议**: 使用成熟的Python SDK，做好沙箱测试

### 风险2: 订单状态一致性
- **风险**: 支付回调、管理员操作、系统超时并发导致状态错乱
- **建议**: 使用数据库事务+状态机验证，记录状态变更日志

### 风险3: 暗号系统并发
- **风险**: 高并发时暗号重复
- **建议**: 预生成暗号池，使用数据库唯一约束

### 风险4: 实时通知可靠性
- **风险**: 飞书/微信推送失败导致用户不知道订单状态
- **建议**: 实现推送重试机制+兜底方案（用户主动刷新）
