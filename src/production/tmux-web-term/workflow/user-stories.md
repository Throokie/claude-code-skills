# 用户故事分析

## WHO (用户角色)

### 主要用户
1. **多项目管理用户**
   - 同时维护多个服务器/项目
   - 需要为不同项目开不同标签页
   - 经常切换不同工作环境

2. **长任务用户**
   - 运行需要长时间保持的进程
   - 需要在多个标签页查看同一会话
   - 关注任务状态不中断

3. **团队协作用户**
   - 多人共享 tmux 会话
   - 各自标签页连接同一会话
   - 需要实时协作和监控

### 次要用户
- 系统管理员：管理多个服务器会话
- 开发者：同时开发多个项目

---

## WHAT (用户需求)

### 核心功能需求
1. **标签页管理**
   - 创建空白标签页（不立即创建 tmux 会话）
   - 标签页可以关联 0 或 1 个 tmux 会话
   - 多个标签页可以连接同一会话

2. **会话选择**
   - 新建标签页后选择要连接的会话
   - 查看服务器上所有可用会话列表
   - 直接在界面创建新 tmux 会话
   - 可以暂时不连接会话

3. **会话切换**
   - 在不关闭标签页的情况下切换会话
   - 保留标签页的快照内容
   - 连接新会话后恢复终端显示

4. **数据迁移**
   - 自动迁移旧版数据到新版格式
   - 保持现有用户数据不丢失

### 使用场景
1. **场景1：新建工作空间**
   - 用户想要为一个新项目创建工作空间
   - 点击"新建标签页"创建空白标签页
   - 在会话选择器中选择或创建会话
   - 开始工作

2. **场景2：多视角监控**
   - 用户运行了一个长时间任务
   - 在标签页 A 中查看任务输出
   - 在标签页 B 中同时查看同一会话的不同部分
   - 两个标签页独立滚动，互不干扰

3. **场景3：团队协作**
   - 团队成员 A 在标签页中连接到共享会话
   - 团队成员 B 也在自己的标签页连接到同一会话
   - 两人可以同时看到相同内容，各自独立操作

## WHY (用户痛点)

### 当前问题
1. **概念混淆**
   - 侧边栏项和 tmux 会话概念混淆
   - 用户无法为同一会话打开多个标签页
   - 必须先创建会话才能使用，流程不灵活

2. **管理不便**
   - 无法灵活组织工作空间
   - 切换会话需要断开重连
   - 无法预览可用会话再决定连接哪个

3. **协作困难**
   - 多人无法独立查看同一会话
   - 会话切换影响所有连接者

### 痛点严重程度
- **高**：概念混淆导致认知负担，影响日常使用
- **高**：无法多标签页查看同一会话，限制工作效率
- **中**：会话管理不灵活，需要频繁断开重连

### 期望结果
- 标签页和会话概念清晰分离
- 可以灵活管理多个工作空间
- 同一会话可以在多个标签页查看
- 新建标签页时可以预览和选择会话
