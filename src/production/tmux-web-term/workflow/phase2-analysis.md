  3.3 核心设计原则

  1. 解耦优先：标签页与tmux会话独立生命周期，支持多对多关系
  2. 渐进增强：先保证基础功能稳定，再逐步添加协作、AI等高级功能
  3. 概念对齐：UI术语与tmux概念保持一致，减少用户认知负担
  4. 状态优先：所有设计以"不丢状态"为第一准则

  4.3 风险提示

  1. 性能风险：50+标签页场景下的前端渲染性能需重点验证
  2. 安全风险：Web终端的命令注入和逃逸风险需严格防范
  3. 兼容性风险：不同浏览器对WebSocket和xterm.js的支持差异
  4. 状态同步风险：网络抖动下的状态一致性保障

  路径和高级功能规划上。建议采用解耦式架构，以tmux Control Mode为核心技术方案，
  分阶段实现基础功能→增强体验→高级特性。
