# Bypass Permissions 模式行为规范

> **版本**: v1.0
> **创建日期**: 2026-03-20
> **适用范围**: bypass permissions 模式下的所有 AI 操作

---

## 核心原则

**在 bypass permissions 模式下，AI 必须在执行以下操作前输出结构化意图声明，等待用户确认。**

---

## 意图声明格式

AI 必须在执行命令前输出以下格式的声明：

```markdown
---
📋 意图声明
**操作类型**: [类型名称]
**风险等级**: ⚠️ 中 / 🔴 高
**执行命令**: `实际命令`
**预期结果**: 描述期望的结果
**潜在影响**: 描述可能的副作用
**回滚方案**: 如何撤销此操作
---

是否继续？[Y/n]
```

---

## 需要声明的操作类型

### 1. 方案变更类 (Plan Change)

**触发场景**:
- 当前方法遇到困难，想尝试其他方案
- 发现更好的替代方法
- 需要回退到之前的状态

**声明示例**:
```markdown
---
📋 意图声明
**操作类型**: 方案变更
**风险等级**: ⚠️ 中
**当前方案**: 使用 grimblast 截图
**替代方案**: 使用 wf-recorder 录制
**变更原因**: grimblast 不支持区域选择
---

是否切换到新方案？[Y/n]
```

**关键词**（出现这些词时必须声明）:
- 放弃、跳过、换方案、换个方法
- bypass、workaround、plan B
- 回退、rollback、revert

---

### 2. 破坏性操作类 (Destructive)

**触发场景**:
- 删除文件或目录
- 覆盖现有文件
- 修改系统配置
- 格式化/分区操作

**声明示例**:
```markdown
---
📋 意图声明
**操作类型**: 破坏性操作
**风险等级**: 🔴 高
**执行命令**: `rm -rf /tmp/old-backup/`
**预期结果**: 删除旧备份目录
**潜在影响**: 数据永久丢失，无法恢复
**回滚方案**: 从最近的备份恢复
**已备份**: /backup/old-backup-20260320.tar.gz
---

⚠️  此操作不可逆！是否继续？[y/N]
```

**触发命令**:
| 命令模式 | 风险等级 |
|---------|---------|
| `rm -rf` | 🔴 高 |
| `rm -r`, `rm -f` | ⚠️ 中 |
| `dd if=` | 🔴 高 |
| `mkfs`, `fdisk` | 🔴 高 |
| `echo ... > file` | ⚠️ 中 |
| `cp -f`, `mv -f` | ⚠️ 中 |
| `chmod -R 777` | 🔴 高 |
| `chown -R` | ⚠️ 中 |

---

### 3. 服务重启类 (Service Restart)

**触发场景**:
- 重启桌面环境
- 重启系统服务
- 重启网络服务

**声明示例**:
```markdown
---
📋 意图声明
**操作类型**: 服务重启
**风险等级**: 🔴 高
**执行命令**: `systemctl restart hyprland`
**预期结果**: 重新加载 Hyprland 配置
**潜在影响**: 当前会话中断，未保存工作丢失
**建议**: 请先保存所有工作
---

⚠️  会话将中断！是否继续？[y/N]
```

**触发命令**:
| 命令模式 | 影响 |
|---------|------|
| `restart hyprland` | 会话中断 |
| `restart sddm/gdm/lightdm` | 退出登录 |
| `reboot`, `shutdown` | 系统重启/关机 |
| `pkill kitty/tmux` | 终端/会话关闭 |

---

### 4. 人工介入类 (Human Required)

**触发场景**:
- 验证码输入
- 二维码扫描
- GUI 交互操作
- 身份认证

**声明示例**:
```markdown
---
📋 意图声明
**操作类型**: 人工介入
**风险等级**: ⚠️ 中
**当前步骤**: 登录飞书
**需要操作**: 扫描二维码
**等待时间**: 约 30 秒
---

📱 请在手机上扫描二维码完成验证

[等待用户操作...]
```

**触发场景**:
- 验证码、二维码
- 登录、认证
- 点击、选择
- 输入密码/Token

---

### 5. 网络修改类 (Network Change)

**触发场景**:
- 修改防火墙规则
- 修改网络配置
- 断开/重连网络

**声明示例**:
```markdown
---
📋 意图声明
**操作类型**: 网络修改
**风险等级**: 🔴 高
**执行命令**: `ufw disable`
**预期结果**: 禁用防火墙
**潜在影响**: 系统暴露在网络中，可能被攻击
**回滚方案**: `ufw enable`
---

⚠️  防火墙将关闭！是否继续？[y/N]
```

---

### 6. Git 危险操作类 (Git Risky)

**触发场景**:
- 强制重置分支
- 删除未提交更改
- 强制推送远程

**声明示例**:
```markdown
---
📋 意图声明
**操作类型**: Git 危险操作
**风险等级**: 🔴 高
**执行命令**: `git reset --hard HEAD~1`
**预期结果**: 回退到上一个 commit
**潜在影响**: 丢失当前工作目录的所有未提交更改
**已备份**: `git stash push -m "backup before reset"`
**回滚方案**: `git stash pop` 或 `git reflog`
---

⚠️  未提交更改将丢失！是否继续？[y/N]
```

---

### 7. Docker 危险操作类 (Docker Risky)

**触发场景**:
- 删除所有未使用资源
- 强制删除运行中容器

**声明示例**:
```markdown
---
📋 意图声明
**操作类型**: Docker 危险操作
**风险等级**: 🔴 高
**执行命令**: `docker system prune -a`
**预期结果**: 清理所有未使用的容器、镜像、网络
**潜在影响**: 正在运行的容器可能受影响
**回滚方案**: 重新 pull 需要的镜像
---

⚠️  将删除所有未使用资源！是否继续？[y/N]
```

## Hook 自动检测规则

### 检测流程

```
用户命令
    ↓
1. 检查是否带意图声明
    ↓ 无声明
2. 分析命令类型
    ↓
3. 匹配风险类别
    ↓
4. 拦截并要求声明
```

### 自动拦截的命令

| 类别 | 命令模式 | 动作 |
|------|---------|------|
| 破坏性 | `rm -rf/*` | 🔴 拦截 |
| 破坏性 | `rm -rf ~/*` | 🔴 拦截 |
| 破坏性 | `mkfs` | 🔴 拦截 |
| 服务重启 | `pkill kitty` | 🔴 拦截 |
| 服务重启 | `reboot` | 🔴 拦截 |
| 网络修改 | `ufw disable` | ⚠️ 拦截 |
| 网络修改 | `iptables -F` | ⚠️ 拦截 |
| Git 危险 | `git reset --hard` | ⚠️ 拦截 |
| Git 危险 | `git push -f` | ⚠️ 拦截 |

## 豁免情况

以下情况**不需要**声明，可以直接执行：

1. **只读操作**: `cat`, `ls`, `grep`, `head`, `tail`
2. **信息收集**: `systemctl status`, `journalctl`, `ps aux`
3. **安全修改**: `git add`, `git commit`
4. **临时文件**: 操作 `/tmp/` 目录
5. **用户明确授权后**: 已确认的操作可以执行后续步骤

---

## 违规处理

如果 AI 在 bypass permissions 模式下**未声明直接执行**：

1. **Hook 自动拦截** - 命令被阻止执行
2. **返回错误消息** - 告知 AI 需要意图声明
3. **记录日志** - 记录违规行为供后续分析

---

## 最佳实践

### 给 AI 的建议

1. **高风险操作前暂停** - 停下来思考是否需要用户确认
2. **提供回滚方案** - 告诉用户如何撤销
3. **说明已备份** - 如果有备份，明确告知
4. **使用确认标志** - `[Y/n]` 表示默认同意，`[y/N]` 表示默认拒绝

### 给用户的建议

1. **审查意图声明** - 不要盲目确认
2. **要求更多信息** - 如果声明不够清楚，要求 AI 补充
3. **使用测试环境** - 危险操作先在测试环境验证

## 配置示例

### settings.json

```json
{
  "permissions": {
    "defaultMode": "bypassPermissions"
  },
  "hooks": {
    "PreToolUse": [{
      "matcher": "Bash",
      "hooks": [{
        "type": "command",
        "command": "fish ~/.claude/hooks/pre-tool-use-bypass-check.sh",
        "timeout": 3
      }]
    }]
  }
}
```

### Hook 脚本

```fish
#!/usr/bin/env fish
# pre-tool-use-bypass-check.sh

set -l input "$CLAUDE_HOOK_INPUT"
set -l command (echo "$input" | jq -r '.tool.command // ""')

# 检查是否带意图声明标记
if not string match -qi "*意图声明*" "$command"
    # 检查是否是危险命令
    if string match -qi "rm -rf" "$command"
        echo '{"approved": false, "reason": "需要意图声明"}'
        exit 1
    end
end

echo '{"approved": true}'
```
