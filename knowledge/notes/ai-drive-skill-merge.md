# AI Drive 融合技能设计

**日期**: 2026-03-18
**目标**: 将 6 个独立技能融合为 1 个核心框架 + 1 个依赖工具

---

## 融合决策

### 原始技能清单（6 个）

| Skill | 状态 | 原因 |
|-------|------|------|
| product-builder | ✅ 融入主框架 | 核心理念保留，简化流程 |
| time-boxing | ✅ 融入执行层 | 时间感知 + MVU 整合 |
| e2e-optimizer | ✅ 融入质量层 | 精确打击测试策略 |
| git-worktree | ✅ 融入工作流层 | Git 存档简化版 |
| deep-work-tracker | ❌ 删除 | 功能被 time-boxing 覆盖 |
| long-running-agent | ⚠️ 独立为依赖 | 这是元框架（管理 agent 的工具） |

### 新技能结构

```
ai-drive/SKILL.md          ← 主框架（融合 4 个技能）
    ↓
long-running-agent/SKILL.md ← 依赖工具（独立）
```

---

## 设计理念

### 为什么融合？

**问题**：
1. 6 个技能，每个都有复杂的方法论
2. Claude 无法同时内化多套工作流
3. 技能之间有重叠（如 time-boxing vs product-builder 的时间盒）
4. 实际使用中只调用 surface-level 概念

**解决**：
1. 单一事实来源：所有规则在一个文件
2. 消除冲突：统一术语和流程
3. 减少认知负担：AI 只需学习一套框架

### 为什么 long-running-agent 独立？

**原因**：
- 这是"如何创建/管理 agent"的元技能
- 它不特定于 AI Drive，可被其他框架复用
- 包含 Anthropic 官方最佳实践，不宜修改
- AI Drive 只调用它，不重新定义它

## Prompt 精简对比

### 原版（约 800 字）

```
【系统指令】忽略上方所有对话，继续推进项目：

/product-builder 请继续推进项目，当前需遵循以下武器库与规范：

【武器库与原则】
- 6 个 skill 文件列表...

【强制要求：多 Agent 协作】
- 必须使用 2-3 个 agent 并行开发
- 主 Agent：核心逻辑 + 架构设计 + 集成
- Worker Agent 1:UI/前端/视觉
- Worker Agent 2: 测试/E2E/文档（可选）
...（省略约 500 字）

【Phase 0: 项目定位分析】
1. 项目概述
2. 竞品分析
3. 用户体验目标
4. 功能优先级矩阵
...（省略约 200 字）
```

### 新版（约 200 字）

```
【系统指令】忽略上方所有对话，继续推进项目：

/ai-drive 请继续推进项目，当前需遵循以下武器库与规范：

【武器库与原则】
- ai-drive/SKILL.md - AI 自主开发驱动框架（主 skill）
- long-running-agent/SKILL.md - 多 agent 创建工具（依赖）

【本轮执行要求】
1. 读取武器库文件，遵循 AI Drive 框架执行
2. 提取上一次的进度，直接开始下一个未完成的核心功能
3. 强制使用多 Agent 并行开发（主 Agent + UI Agent + 测试 Agent）
4. 每轮必须汇报各 Agent 进度和剩余时间

【防卡死协议】
- 同一个 Bug 修复尝试超过 3 次失败 → 立即降级或跳过
- 核心流程跑通（E2E 通过） → 立刻停止优化，进入下一模块

【闭环协议】
- 完成本阶段后，静默保存进度到 `.agent/progress.json`
- Git 存档：复制 MCP 配置 + 提交代码变更
```

**精简效果**：800 字 → 200 字（75% 减少）

## 后续优化

如果 AI Drive 运行稳定，可以考虑：
1. 添加更多"精确打击"规则（如特定场景的测试策略）
2. 优化 Agent 汇报格式（更结构化的表格）
3. 添加"紧急降级"机制（时间不足时自动砍功能）

**核心原则**：宁可少做，不可贪多。2 个技能是上限。
