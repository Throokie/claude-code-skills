# 反思任务：从历史经验教训中提取通用原则

## 任务背景

我整理了系统中的历史经验和教训文档，包含 20+ 个错误模式、8 个核心原则、以及多个技术教训。

## 输入材料

### 错误模式库（20+ 个）

#### 系统环境类
- EP-001: root 服务无用户环境变量 → 显式设置 `WAYLAND_DISPLAY` `XDG_RUNTIME_DIR`
- EP-002: Hyprland 路径大小写 → 用 `bindl` 代替 `bindd` 执行命令
- EP-003: 符号链接路径混乱 → 用绝对路径创建软连接
- EP-004: Chrome Profile 丢失 → 用持久化目录
- EP-024: tmux 缓冲区 UTF-8 截断 → 定期 `tmux clear-history`

#### 配置类
- EP-005: keyd 配置冲突 → 参考 git 历史，重启服务
- EP-006: OpenClaw 权限隔离 → 配置目录移到服务用户目录
- EP-007: OpenSnitch 默认策略 → 设置 `DefaultAction: deny`
- EP-025: hypridle D-Bus 抑制冲突 → `ignore_dbus_inhibit = true`

#### 工具使用类
- EP-008: MCP 调用失败沉默 → 明确告知失败原因 + 问替代方案
- EP-009: 有 MCP 不用 → MCP 优先，Skill 后备
- EP-010: 不读日志瞎分析 → 先解决权限错误，再分析其他
- EP-019: Skill 软连接断裂 → 用绝对路径，包含完整路径层次

#### 应用类
- EP-011: fcitx5 词库丢失 → 检查配置链接
- EP-012: tmux 按键冲突 → 检查 tmux 配置和系统绑定
- EP-013: Skill 目录软连接断裂 → 用绝对路径
- EP-021: tmux paste 后回车不生效 → 使用 `C-m` 代替 `Enter`
- EP-022: 波浪号路径展开失败 → `${VAR#\~/}` 单反斜杠
- EP-023: AI 空闲检测误判 → 先检测内容变化，再检测 AI 状态

#### AI/API 类
- EP-026: DashScope video 误解 → 实际是关键帧序列 + fps
- EP-027: Neo 扩展 Service Worker 停止 → 添加 chrome.alarms 心跳
- EP-028: 抖音图片下载 403 → 带 Referer 防盗链

### 核心原则（8 个领域）

1. **调试优先级**：权限错误 > 服务状态 > 配置语法 > 网络连接 > 业务逻辑
2. **MCP 工具优先**：用户请求 → 识别意图 → 优先 MCP → MCP 无法满足 → 降级 Skill
3. **目录结构规范**：dotfiles/src/projects/data/backups
4. **多用户环境意识**：服务用户隔离，配置目录在服务用户家目录
5. **Skill 管理规范**：完成后创建软链接，用绝对路径
6. **tmux 自动化**：定期清理缓冲区，使用 C-m，添加延迟
7. **AI Drive 融合工作流**：多 agent 并行，每轮汇报，防卡死协议
8. **技能精简原则**：上限 2 个技能，Prompt 控制在 200 字

### 关键洞察

从 281 条历史对话中提炼：
- 高频问题反映系统复杂性（OpenClaw 部署 24 次，键盘配置 13 次）
- 错误模式可跨场景复用
- 规范性问题需要强制性机制
- 技能融合优于堆砌
- 失败透明化建立信任
