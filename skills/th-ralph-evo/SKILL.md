## 核心概念

### 1. 决策驱动开发 (Decision-Driven Development)

```
用户输入 (零散需求/抱怨)
    │
    ▼
┌────────────────────┐
│  decision-record   │  ← 自动触发，无需等待用户完整描述
│  需求演进与追踪    │
└────────┬───────────┘
         │
         ▼
    ┌────────────┐
    │ 功能主题   │  ← AI主动归纳主题
    ├────────────┤
    │ 1级解读    │  ← 表面层方案
    │ 2级解读    │  ← 系统层方案
    │ N级解读    │  ← 深层剖析
    │ 本地意见   │  ← 代码库分析
    │ 外部意见   │  ← 多模型聚合
    │ 最终方案   │  ← 用户确认
    └─────┬──────┘
          │
          ▼
    ┌────────────┐
    │ PRD 生成   │  ← 自动转换为 PRD
    └────────────┘
```

### 2. 自动化触发机制

```yaml
触发条件:
  decision_record:
    - 文件变更监听: ~/src/docs/decisions/*.md
    - 状态变化: 草稿 → 已确认
    - 自动动作: 启动 deep-coder2 调研

  prd_evolution:
    - PRD 状态: 已确认 → 待实现
    - 自动动作: 创建 prd-to-code 团队

  ralpha:
    - PRD 就绪信号
    - 自动动作: 创建 worktree 并行开发

  bug_analyzer:
    - 测试失败信号
    - 代码覆盖率低于阈值
    - 自动动作: 分析影响并修复
```

## 工作流程

### Phase 1: 启动自动化开发

**用户输入**:
```
启动自动化开发 - 用户认证模块
```

**系统自动执行**:

```
1. 检查决策记录
   ├── 读取 ~/src/docs/decisions/INDEX.md
   ├── 查找相关决策记录
   └── 若无，询问是否创建

2. 检查 PRD 状态
   ├── 读取 prd-evolution tree.md
   ├── 确认是否有待实现 PRD
   └── 若无，从 decision-record 生成

3. 启动深度调研 (可选)
   ├── 触发 th-deep-coder2
   ├── 收集技术方案
   └── 整合到 PRD

4. 创建实现团队
   ├── 使用 orchestrator 创建团队
   ├── 根据 PRD 技术栈选择代理类型
   └── 分配 worktree

5. 启动可视化面板
   └── 打开 dashboard 显示实时状态
```

### Phase 3: Bug 分析与闭环

```
测试失败触发:

1. bug-analyzer 分析
   ├── 读取测试失败日志
   ├── 使用 GitNexus 分析影响范围
   ├── 定位根因
   └── 生成修复方案

2. 决策分支
   ├── 若影响小: 直接修复
   ├── 若影响大: 创建 decision-record 讨论
   └── 若需架构调整: 更新 PRD-evolution

3. 自动修复执行
   ├── 创建 hotfix worktree
   ├── 实施修复
   ├── 回归测试
   └── 合并或废弃

4. 学习记录
   ├── 记录 bug 模式到知识库
   ├── 更新测试用例
   └── 反馈到 decision-record
```

### 与现有技能的关系

```

        │
        ▼ 自动触发

        │
        ▼ 数据流入

        │

        │

        │

        │
        └───────┐
```

### 配置示例

```yaml
# workflow-config.yaml
workflow:
  name: "用户认证模块开发"
  triggers:
    decision_record:
      path: "~/src/docs/decisions/user-auth.md"
      auto_investigate: true

  prd:
    id: "prd-user-auth-v1"
    auto_create: true
    evolution:
      enable_version_tree: true

  team:
    type: "multi-agent"
    workers:
      - name: "data-agent"
        layer: "data"
        model: "typescript-reviewer"
      - name: "logic-agent"
        layer: "logic"
        model: "typescript-reviewer"
      - name: "ui-agent"
        layer: "ui"
        model: "typescript-reviewer"
      - name: "test-agent"
        layer: "test"
        model: "tdd-guide"

  quality_gates:
    test_coverage: 90
    security_scan: true
    code_review: true

  feedback:
    auto_bug_analyze: true
    create_decision_on_failure: true
```

## 快速开始

### 1. 启动一个完整的自动化开发流程

```bash
# 启动 Ralph Evolution
ralph-evo start --project user-auth --from-decision

# 或从现有 PRD 开始
ralph-evo start --project user-auth --from-prd prd-user-auth-v1
```

### 2. 在 Claude Code 中使用

```
用户: 启动自动化开发 - 实现用户认证系统

AI:
1. 读取 decision-record 中关于用户认证的决策
2. 检查 PRD 状态
3. 启动深度调研 (询问用户是否进行)
4. 创建多代理团队
5. 打开可视化面板
6. 开始并行开发
```

---

## 注意事项

1. **自动 vs 手动**: 系统支持全自动化和半自动化模式，关键节点会询问用户确认
2. **资源管理**: 自动清理已完成的 worktree，监控磁盘使用
3. **错误处理**: 遇到关键错误时暂停，生成决策记录供用户审阅
4. **学习积累**: Bug 分析结果自动记录到知识库，避免重复问题
