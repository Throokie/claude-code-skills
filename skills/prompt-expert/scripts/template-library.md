## 功能开发

### 模板 3: REST API 端点

```
你正在实现一个 REST API 端点。

**端点信息**:

- 路径：{endpointPath}

**技术要求**:

    "total": 100
  }

```

**测试要求**:
- 单元测试（成功场景）
- 单元测试（失败场景）
- 集成测试
- 认证测试

**工作流程**:
1. 定义请求/响应 schema
2. 编写测试用例
3. 实现端点逻辑
4. 添加错误处理
5. 运行测试并验证
```

### 模板 4: 用户认证功能

```
你正在实现用户认证功能。

**认证方式**: JWT Token

**功能列表**:
- [ ] 用户注册（邮箱 + 密码）
- [ ] 用户登录
- [ ] Token 刷新
- [ ] 密码重置
- [ ] 邮箱验证

**安全要求**:
- 密码使用 bcrypt 加密（cost factor >= 12）
- JWT 签名密钥至少 256 位
- Token 过期时间：access=15min, refresh=7d
- 防止暴力破解（rate limiting）
- HTTPS only

**数据库表**:
- users (id, email, password_hash, created_at, updated_at)
- refresh_tokens (id, user_id, token, expires_at, revoked)

**工作流程**:
1. 设计数据库 schema
2. 实现密码哈希工具
3. 实现注册端点
4. 实现登录端点
5. 实现 token 管理
6. 添加中间件验证
```

## 代码审查

### 模板 7: 功能代码审查

```
你正在进行代码审查。

**审查范围**: {filesChanged}

**审查清单**:

### 1. 功能正确性
- [ ] 实现了所有需求
- [ ] 边界情况已处理
- [ ] 错误处理适当

### 2. 代码质量
- [ ] 命名清晰有意义
- [ ] 函数职责单一（< 50 行）
- [ ] 无重复代码（DRY）
- [ ] 适当的注释（解释 why 而非 what）

### 3. 安全性
- [ ] 输入已验证
- [ ] 无硬编码密钥
- [ ] SQL 使用参数化查询
- [ ] XSS 防护

### 4. 性能
- [ ] 无 N+1 查询
- [ ] 适当的索引
- [ ] 资源正确释放

### 5. 测试
- [ ] 单元测试覆盖核心逻辑
- [ ] 集成测试验证流程
- [ ] 测试数据合理

**问题分级标准**:
- 🔴 CRITICAL: 安全漏洞、数据丢失风险
- 🟠 HIGH: 功能缺陷、严重性能问题
- 🟡 MEDIUM: 代码质量问题、可维护性隐患
- 🟢 LOW: 风格问题、轻微优化建议

**输出格式**: 按文件列出发现的问题
```

## 使用指南

### 如何使用模板

1. **选择模板**: 根据场景选择最匹配的模板
2. **填充变量**: 替换 `{variable}` 为你的具体信息
3. **调整细节**: 根据项目特点调整要求
4. **添加上下文**: 补充项目特定的约束和约定

### 模板组合

可以组合多个模板使用：

```
使用模板 3（REST API 端点）实现功能
完成后使用模板 7（功能代码审查）进行自查
最后使用模板 13（API 文档）编写文档
```
