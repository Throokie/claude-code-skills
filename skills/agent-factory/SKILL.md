# Agent Factory Skill - 母体触发器

## 触发条件

当用户表达以下意图时触发：

- "这个项目很复杂，需要专业团队"
- "帮我生成处理这个任务的 Agent"
- "创建定制化的 Agent 团队"
- "需要动态生成专业 Agent"
- "这个项目需要 [前端/后端/算法] 专家"

## 工作流程

```
用户描述项目需求
    ↓
提取关键信息（技术栈、约束、版本）
    ↓
Spawn Agent Factory（母体）
    ↓
母体分析并生成 Agent 配置
    ↓
输出可执行的 Agent 创建命令
    ↓
用户确认后执行
```

## 信息收集模板

向用户询问（如果未提供）：

```yaml
1. 项目描述: [一句话描述要做什么]
2. 技术栈:
   - 前端: [框架/语言]
   - 后端: [框架/语言]
   - 数据库: [类型]
3. 版本号: [语义化版本]
4. 特殊需求:
   - [性能/安全/特殊功能要求]
5. 约束条件:
   - [时间/资源/技术限制]
```

## 母体调用

```javascript
Agent({
  subagent_type: "general-purpose",
  name: "agent-factory",
  prompt: `
    作为 Agent Factory（母体），请分析以下项目并生成专业 Agent 团队配置：

    项目信息:
    ${collected_info}

    要求:
    1. 分析项目复杂度
    2. 确定需要的 Agent 数量和类型
    3. 为每个 Agent 生成定制化系统提示词
    4. 设计 Agent 间的协作关系
    5. 输出可执行的启动命令

    请按照 ~/.claude/agents/agent-factory.md 中的格式输出完整配置。
  `
})
```

## 输出示例

母体应输出：

1. **项目分析报告** - 复杂度评估和团队构成建议
2. **Agent 配置清单** - 每个 Agent 的详细配置
3. **系统提示词** - 定制化的 Agent 提示词
4. **启动命令** - 可直接执行的命令
