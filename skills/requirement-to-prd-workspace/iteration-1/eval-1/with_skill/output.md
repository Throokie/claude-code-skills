## 2. 产品概述

### 2.1 产品背景
小型团队在任务协作中常面临任务分配不明确、进度跟踪困难、优先级混乱等问题。现有工具要么过于复杂（如 Jira），要么功能不足（如简单待办应用）。本产品旨在为 5-20 人的小型团队提供一个轻量级、易上手的任务管理解决方案。

### 2.2 产品目标
- 帮助团队建立清晰的任务分配机制，减少沟通成本
- 提供直观的任务状态视图，提升团队协作效率
- 支持优先级和截止日期的灵活管理，确保重要任务按时完成
- 实现系统响应时间小于 500ms，页面加载时间小于 2s

### 2.3 目标用户
| 用户类型 | 特征 | 主要需求 |
|---------|------|---------|
| 团队管理者 | 5-20人团队负责人 | 快速分配任务、监控整体进度、查看团队负载 |
| 团队成员 | 开发者、设计师、运营等 | 清晰查看个人任务、标记完成状态、设置提醒 |
| 项目负责人 | 同时管理多个项目 | 跨项目任务视图、项目进度概览 |

### 3.2 详细功能描述

#### F-001: 用户登录与注册
**用户故事**: 作为团队成员，我想要注册和登录系统，以便访问我的任务。

**验收标准**:
- [ ] 用户可以使用邮箱和密码注册账号
- [ ] 用户可以使用邮箱和密码登录
- [ ] 密码需要加密存储
- [ ] 登录后保持会话状态（JWT Token）
- [ ] 支持登出功能

**输入/输出**:
- 输入: 邮箱、密码
- 输出: 用户信息、认证令牌

**边界情况**:
- 邮箱已注册: 提示"该邮箱已被注册"
- 密码强度不足: 提示"密码需包含至少8位字符，包含字母和数字"
- 登录失败: 提示"邮箱或密码错误"（不区分）

---

#### F-002: 任务创建
**用户故事**: 作为团队成员，我想要创建新任务，以便记录需要完成的工作。

**验收标准**:
- [ ] 可以输入任务标题（必填，最多100字符）
- [ ] 可以输入任务描述（选填，最多2000字符）
- [ ] 可以选择任务优先级（高/中/低，默认中）
- [ ] 可以设置截止日期（选填，日期选择器）
- [ ] 创建后任务状态为"待办"

**输入/输出**:
- 输入: 标题、描述、优先级、截止日期
- 输出: 创建成功的任务对象

**边界情况**:
- 标题为空: 提示"请输入任务标题"
- 标题超过100字符: 截断或提示"标题过长"
- 截止日期早于今天: 提示"截止日期不能早于今天"

---

#### F-003: 任务分配
**用户故事**: 作为团队管理者，我想要将任务分配给特定成员，以便明确责任。

**验收标准**:
- [ ] 创建任务时可以选择负责人（团队成员列表）
- [ ] 可以修改已创建任务的负责人
- [ ] 被分配人收到通知
- [ ] 支持将任务分配给自己

**输入/输出**:
- 输入: 任务ID、被分配人ID
- 输出: 更新后的任务对象

**边界情况**:
- 分配给自己已负责的任务: 静默处理
- 分配不存在的用户: 提示"用户不存在"
- 任务已完成时重新分配: 允许但标记为特殊操作

---

#### F-004: 优先级设置
**用户故事**: 作为团队成员，我想要设置任务优先级，以便区分任务的重要程度。

**验收标准**:
- [ ] 支持三种优先级：高（红色）、中（黄色）、低（灰色）
- [ ] 创建时可设置，创建后可修改
- [ ] 任务列表支持按优先级排序
- [ ] 高优先级任务在列表中醒目显示

**输入/输出**:
- 输入: 优先级级别（high/medium/low）
- 输出: 更新后的任务对象

**边界情况**:
- 不设置优先级: 默认为"中"

---

#### F-005: 截止日期管理
**用户故事**: 作为团队成员，我想要设置任务截止日期，以便合理安排时间。

**验收标准**:
- [ ] 支持日期选择器选择截止日期
- [ ] 支持清除已设置的截止日期
- [ ] 即将到期（24小时内）的任务有特殊标记
- [ ] 已过期的任务显示过期状态

**输入/输出**:
- 输入: 日期（YYYY-MM-DD 格式）
- 输出: 更新后的任务对象

**边界情况**:
- 截止日期为今天: 标记为"今天到期"
- 截止日期已过: 标记为"已过期"，但仍可操作
- 不设置截止日期: 显示为"无截止日期"

---

#### F-006: 任务状态流转
**用户故事**: 作为团队成员，我想要更新任务状态，以便团队了解进度。

**验收标准**:
- [ ] 支持三种状态：待办、进行中、已完成
- [ ] 可以通过点击或拖拽改变状态
- [ ] 状态变更记录操作人
- [ ] 任务完成后可以重新打开

**输入/输出**:
- 输入: 状态值（todo/in_progress/done）
- 输出: 更新后的任务对象

**边界情况**:
- 已完成任务重新打开: 状态变为"待办"
- 批量状态变更: 支持选择多个任务批量操作

## 4. 非功能需求

### 4.1 性能要求
- 页面首次加载时间 < 2秒
- 任务列表查询响应时间 < 500ms
- 支持同时在线用户 50 人
- 支持单个用户最多 1000 个任务

### 4.2 安全要求
- 用户密码使用 bcrypt 加密存储
- API 使用 JWT 认证，Token 有效期 7 天
- 所有 API 请求进行身份验证
- 防止 XSS 和 CSRF 攻击
- 用户只能访问自己有权限的数据

### 4.3 兼容性要求
- 支持 Chrome、Firefox、Safari、Edge 最新两个版本
- 响应式设计，支持桌面端（1280px+）和平板端（768px+）
- 移动端提供基本功能浏览（可选）

### 4.4 可用性要求
- 系统可用性 99.5% 以上
- 数据每日自动备份
- 支持数据导出（CSV/JSON 格式）

## 5. 技术方案

### 5.1 技术栈
| 层级 | 技术选型 | 说明 |
|------|---------|------|
| 前端 | React 18 + TypeScript | 组件化开发，类型安全 |
| 状态管理 | Zustand | 轻量级状态管理 |
| UI 组件库 | Ant Design 或 Chakra UI | 丰富的企业级组件 |
| 后端 | Node.js + Express | 轻量级 Web 框架 |
| 数据库 | PostgreSQL | 关系型数据库，支持复杂查询 |
| ORM | Prisma | 类型安全的数据库访问 |
| 认证 | JWT (jsonwebtoken) | 无状态认证方案 |
| 文件存储 | 本地文件系统 / AWS S3 | 附件存储 |

### 5.3 数据模型

**User（用户）**
```
id: UUID (PK)
email: String (unique)
password: String (hashed)
name: String
avatar: String (URL, optional)
createdAt: DateTime
updatedAt: DateTime
```

**Task（任务）**
```
id: UUID (PK)
title: String
description: Text (optional)
status: Enum (todo/in_progress/done)
priority: Enum (high/medium/low)
dueDate: DateTime (optional)
assigneeId: UUID (FK -> User, optional)
creatorId: UUID (FK -> User)
createdAt: DateTime
updatedAt: DateTime
```

**Comment（评论）**
```
id: UUID (PK)
content: Text
taskId: UUID (FK -> Task)
authorId: UUID (FK -> User)
createdAt: DateTime
```

**Attachment（附件）**
```
id: UUID (PK)
filename: String
url: String
size: Int
mimeType: String
taskId: UUID (FK -> Task)
uploadedById: UUID (FK -> User)
createdAt: DateTime
```

**Notification（通知）**
```
id: UUID (PK)
type: Enum (task_assigned/due_reminder/comment_added)
content: String
isRead: Boolean
userId: UUID (FK -> User)
taskId: UUID (FK -> Task, optional)
createdAt: DateTime
```

## 6. UI/UX 设计

### 6.1 设计原则
- **简洁清晰**: 界面元素精简，重点突出任务内容
- **高效操作**: 常用操作不超过两次点击
- **视觉反馈**: 状态变化有明显视觉提示
- **一致性**: 交互模式保持一致，降低学习成本

### 6.2 关键页面
| 页面 | 描述 | 主要元素 |
|------|------|---------|
| 登录页 | 用户认证入口 | 邮箱输入框、密码输入框、登录按钮、注册链接 |
| 注册页 | 新用户注册 | 邮箱输入框、密码输入框、确认密码、注册按钮 |
| 任务列表页 | 核心工作区 | 视图切换、筛选栏、任务列表/看板、新建任务按钮 |
| 任务详情页 | 查看/编辑任务 | 任务表单、评论列表、附件列表、操作按钮 |
| 统计页 | 数据概览 | 数据卡片、图表、筛选器 |
| 个人设置页 | 用户信息管理 | 头像上传、个人信息、通知设置 |

### 6.3 交互流程

**创建任务流程**
```
1. 点击"新建任务"按钮
   ↓
2. 弹出创建表单（标题、描述、优先级、截止日期、负责人）
   ↓
3. 填写信息，点击"创建"
   ↓
4. 表单验证
   ↓
5. 创建成功，任务出现在列表中
```

**任务状态流转**
```
待办 --(开始)--> 进行中 --(完成)--> 已完成
  ↑                                      |
  └-----------(重新打开)-----------------┘
```

**看板视图拖拽**
```
1. 在看板视图下，鼠标按住任务卡片
   ↓
2. 拖动到目标状态列
   ↓
3. 释放鼠标，任务状态更新
   ↓
4. 显示更新成功提示
```

| 截止日期 (Due Date) | 任务需要完成的最后日期 |
| 负责人 (Assignee) | 被指派完成任务的团队成员 |
