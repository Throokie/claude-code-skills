## 2. 产品概述

### 2.1 产品背景

小型团队在任务管理方面面临以下痛点：
- 任务分配不明确，责任边界模糊
- 任务进度难以追踪，缺乏可视化
- 优先级混乱，重要任务被淹没
- 截止日期管理混乱，经常出现延误

### 2.2 产品定位

TaskFlow 是一款面向小型团队的轻量级任务管理系统，旨在提供简单直观的任务创建、分配、追踪和管理功能，帮助团队提升协作效率。

### 2.3 目标用户画像

- **团队规模**：5-50人
- **行业类型**：互联网初创公司、设计工作室、咨询团队、小型开发团队
- **使用场景**：敏捷开发、项目执行、日常任务追踪
- **技术能力**：具备基础办公软件使用能力，对协作工具有基本认知

## 3. 功能需求

### 3.1 核心功能模块

#### 3.1.1 任务管理模块

##### FR-001: 任务创建
- **描述**：用户可以创建新任务
- **输入字段**：
  - 任务标题（必填，最大100字符）
  - 任务描述（选填，最大2000字符，支持富文本）
  - 任务优先级（必填，选项：紧急/高/中/低，默认：中）
  - 截止日期（选填，日期选择器）
  - 任务负责人（选填，从团队成员列表选择）
  - 标签（选填，多选，支持自定义标签）
- **验证规则**：
  - 标题不能为空且长度不超过100字符
  - 截止日期不能早于当前日期
- **输出**：创建成功后在任务列表中显示

##### FR-002: 任务编辑
- **描述**：任务创建者或负责人可以修改任务信息
- **可编辑字段**：同任务创建
- **权限控制**：
  - 任务创建者：完全编辑权限
  - 任务负责人：可修改状态、进度、评论
  - 其他成员：仅可评论

##### FR-003: 任务删除
- **描述**：任务创建者可删除任务
- **确认机制**：删除前需二次确认
- **数据保留**：删除的任务进入回收站，保留30天后永久删除

##### FR-004: 任务状态管理
- **状态流转**：待处理 → 进行中 → 审核中 → 已完成
- **状态说明**：
  - 待处理：任务已创建但未开始
  - 进行中：负责人已开始处理
  - 审核中：任务完成等待审核
  - 已完成：任务已验收通过
  - 已取消：任务被取消（终态）

##### FR-005: 任务分配
- **描述**：将任务分配给团队成员
- **分配方式**：
  - 创建时直接分配
  - 编辑时重新分配
  - 拖拽分配（看板视图）
- **通知机制**：被分配人收到系统通知

#### 3.1.2 优先级管理模块

##### FR-006: 优先级设置
- **优先级等级**：
  - 紧急（红色）：需要立即处理，24小时内完成
  - 高（橙色）：重要且紧急，3天内完成
  - 中（黄色）：重要但不紧急，1周内完成
  - 低（绿色）：不重要，可延后处理
- **优先级标识**：任务卡片颜色编码 + 优先级标签

##### FR-007: 优先级排序
- **排序方式**：
  - 默认：按优先级降序 + 截止日期升序
  - 自定义：支持用户自定义排序规则
- **筛选功能**：可按优先级筛选任务列表

#### 3.1.3 截止日期管理模块

##### FR-008: 截止日期设置
- **日期格式**：YYYY-MM-DD
- **时间选项**：支持设置具体时间点（选填）
- **重复任务**：支持设置周期性重复任务（周/月/季度）

##### FR-009: 到期提醒
- **提醒时间**：
  - 截止日期前24小时
  - 截止日期前1小时
  - 截止时刻
- **提醒方式**：
  - 系统内通知
  - 邮件通知（可配置）
- **逾期处理**：逾期任务自动标记为红色高亮

#### 3.1.4 任务视图模块

##### FR-010: 列表视图
- **展示字段**：任务标题、负责人、优先级、截止日期、状态
- **排序功能**：支持按任意字段排序
- **分页**：每页默认显示20条，支持调整

##### FR-011: 看板视图
- **列设置**：按状态分列（待处理/进行中/审核中/已完成）
- **拖拽操作**：支持拖拽任务卡片改变状态
- **快速操作**：在看板上可直接编辑任务基本信息

##### FR-012: 日历视图
- **展示方式**：月视图/周视图
- **任务展示**：在对应截止日期显示任务
- **快速创建**：点击日期可直接创建该日期的任务

#### 3.1.5 团队协作模块

##### FR-013: 任务评论
- **评论功能**：团队成员可对任务添加评论
- **评论格式**：支持富文本、@提及成员
- **通知机制**：被@成员收到通知
- **评论历史**：保留完整评论记录

##### FR-014: 任务关注
- **关注功能**：成员可关注感兴趣的任务
- **关注通知**：任务状态变更时通知关注者
- **我的关注**：独立视图查看所有关注的任务

##### FR-015: 活动日志
- **记录内容**：任务创建、编辑、状态变更、分配变更
- **展示方式**：时间线形式展示
- **可见性**：团队成员可见

### 3.2 用户管理模块

#### FR-016: 用户角色
- **系统管理员**：
  - 管理团队成员
  - 配置系统设置
  - 查看所有任务统计
- **项目经理**：
  - 创建和分配任务
  - 查看团队任务
  - 生成项目报告
- **普通成员**：
  - 查看分配给自己的任务
  - 创建个人任务
  - 更新任务状态

#### FR-017: 团队成员管理
- **邀请成员**：通过邮箱邀请加入团队
- **角色分配**：为成员分配系统角色
- **成员列表**：查看团队所有成员及其任务统计

### 3.3 报表统计模块

#### FR-018: 个人任务统计
- **统计维度**：
  - 本周/本月完成任务数
  - 待处理任务数
  - 逾期任务数
  - 任务完成率
- **可视化**：饼图、柱状图展示

#### FR-019: 团队任务统计
- **统计维度**：
  - 团队整体任务完成情况
  - 成员任务分布
  - 项目进度追踪
  - 平均完成时间
- **导出功能**：支持导出Excel/CSV

#### 5.3.2 核心表结构

**users 表**
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    name VARCHAR(100) NOT NULL,
    role VARCHAR(20) DEFAULT 'member' CHECK (role IN ('admin', 'manager', 'member')),
    avatar_url VARCHAR(500),
    timezone VARCHAR(50) DEFAULT 'UTC',
    notification_settings JSONB DEFAULT '{}',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);
```

**tasks 表**
```sql
CREATE TABLE tasks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    title VARCHAR(200) NOT NULL,
    description TEXT,
    creator_id UUID REFERENCES users(id) ON DELETE SET NULL,
    assignee_id UUID REFERENCES users(id) ON DELETE SET NULL,
    project_id UUID REFERENCES projects(id) ON DELETE CASCADE,
    priority VARCHAR(10) DEFAULT 'medium' CHECK (priority IN ('urgent', 'high', 'medium', 'low')),
    status VARCHAR(20) DEFAULT 'todo' CHECK (status IN ('todo', 'in_progress', 'review', 'done', 'cancelled')),
    due_date TIMESTAMP WITH TIME ZONE,
    tags TEXT[],
    attachments JSONB DEFAULT '[]',
    created_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT CURRENT_TIMESTAMP
);

CREATE INDEX idx_tasks_assignee ON tasks(assignee_id);
CREATE INDEX idx_tasks_status ON tasks(status);
CREATE INDEX idx_tasks_due_date ON tasks(due_date);
CREATE INDEX idx_tasks_priority ON tasks(priority);
```

## 6. UI/UX设计

- **简洁优先**：减少视觉噪音，突出任务内容

### 6.4 交互设计

#### 6.4.1 任务创建流程

```
用户点击"新建任务" → 打开创建弹窗
       ↓
用户填写任务信息
       ↓
点击"创建" → 验证输入
       ↓
  ┌─────────┴─────────┐
  ↓                   ↓
验证通过           验证失败
  ↓                   ↓
提交API请求        显示错误提示
  ↓                   ↓
创建成功 ←────────────┘
  ↓
弹窗关闭 + 列表刷新 + 成功提示
```

#### 6.4.2 拖拽排序交互

- **触发**：用户在看板视图拖拽任务卡片
- **反馈**：
  - 拖拽时：卡片半透明，跟随鼠标移动
  - 悬停目标列：列背景高亮
  - 放置后：卡片以动画形式进入新位置
- **结果**：任务状态自动更新，发送API请求

| J/K | 向下/向上选择任务 |
