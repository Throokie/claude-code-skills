## 2. 产品概述

### 2.1 产品背景

随着移动互联网的普及，用户对于个性化服务和数据管理的需求日益增长。本 App 旨在为用户提供一个安全、便捷的个人中心，整合身份认证、资料管理、消息通知和数据导出等核心功能，提升用户体验和数据掌控感。

### 2.2 产品目标

- **目标 1**: 提供流畅的手机号验证码登录体验，确保账户安全的同时降低使用门槛
- **目标 2**: 构建完善的个人资料管理体系，支持用户自主管理个人信息
- **目标 3**: 建立高效的消息通知系统，确保重要信息及时触达用户
- **目标 4**: 实现便捷的数据导出功能，赋予用户数据自主权
- **目标 5**: 达成 App Store / Google Play 评分 4.5+，日活跃用户留存率 60%+

### 2.3 目标用户

| 用户类型 | 特征 | 主要需求 |
|---------|------|---------|
| 普通用户 | 18-45岁，熟悉智能手机操作 | 快速登录、简单管理个人信息 |
| 商务用户 | 25-40岁，注重效率与数据安全 | 数据导出、消息提醒及时性 |
| 隐私敏感用户 | 各年龄段，关注数据主权 | 完整的数据导出、透明的隐私设置 |

### 3.2 详细功能描述

#### F-001: 手机号验证码登录

**用户故事**: 作为用户，我想要使用手机号验证码快速登录，以便无需记忆密码就能安全访问我的账户

**验收标准**:
- [ ] 支持中国大陆手机号格式验证（11位，1开头）
- [ ] 验证码发送间隔不少于 60 秒
- [ ] 验证码有效期为 5 分钟
- [ ] 单日同一手机号发送验证码上限为 10 次
- [ ] 支持登录状态保持（可选 7 天/30 天）
- [ ] 登录失败 5 次后锁定账户 30 分钟

**输入/输出**:
- 输入: 手机号（11位数字）、验证码（6位数字）
- 输出: 登录成功返回用户 Token、用户基础信息；失败返回错误码及提示

**边界情况**:
- 无效手机号: 前端实时校验，提示"请输入正确的手机号"
- 验证码过期: 提示"验证码已过期，请重新获取"
- 频繁请求: 提示"请求过于频繁，请稍后再试"
- 网络异常: 提供重试机制，提示网络连接问题

#### F-002: 个人资料管理

**用户故事**: 作为用户，我想要管理我的个人资料，以便保持信息的准确性和时效性

**验收标准**:
- [ ] 支持查看头像、昵称、手机号、邮箱、生日、性别等基本信息
- [ ] 支持修改昵称（2-20个字符，支持中英文）
- [ ] 支持更换头像（拍照、相册选择，支持裁剪，限制 5MB）
- [ ] 支持绑定/更换邮箱（需验证）
- [ ] 支持修改生日、性别信息
- [ ] 资料修改实时保存，提供操作成功/失败反馈

**输入/输出**:
- 输入: 用户资料字段（昵称、头像文件、邮箱、生日、性别等）
- 输出: 更新后的用户资料、操作结果状态

**边界情况**:
- 昵称重复: 允许重复，不做唯一性校验
- 头像过大: 前端压缩或提示"图片大小超过限制"
- 邮箱格式错误: 实时校验，提示"请输入正确的邮箱格式"
- 网络中断: 本地缓存，待网络恢复后自动同步或提示用户手动重试

#### F-003: 消息通知

**用户故事**: 作为用户，我想要及时接收各类通知消息，以便不错过重要信息

**验收标准**:
- [ ] 支持系统通知、活动通知、账户安全通知等分类
- [ ] 消息列表支持按类型筛选
- [ ] 未读消息数字角标显示
- [ ] 支持消息标记已读/未读
- [ ] 支持消息删除（单条/批量）
- [ ] 支持推送通知（Push Notification）
- [ ] 消息保留 90 天，过期自动清理

**输入/输出**:
- 输入: 消息筛选条件、操作指令（标记已读、删除）
- 输出: 消息列表、操作结果、未读数量

**边界情况**:
- 消息为空: 显示空状态插图和引导文案
- 消息过多: 分页加载，每页 20 条
- 推送权限关闭: 引导用户开启推送权限
- 消息加载失败: 提供重试按钮

#### F-004: 数据导出

**用户故事**: 作为用户，我想要导出我的个人数据，以便备份或在其他平台使用

**验收标准**:
- [ ] 支持导出个人资料数据（JSON/CSV 格式可选）
- [ ] 支持导出消息历史记录
- [ ] 支持导出操作日志（登录、修改记录等）
- [ ] 导出文件通过邮件发送下载链接（有效期 7 天）
- [ ] 导出请求 24 小时内处理完成
- [ ] 每月最多导出 3 次

**输入/输出**:
- 输入: 导出数据类型选择、格式选择、邮箱地址
- 输出: 导出任务提交成功提示、邮件发送确认

**边界情况**:
- 无数据可导出: 提示"暂无可导出数据"
- 邮箱无效: 校验邮箱格式，提示"请输入正确的邮箱"
- 导出频率超限: 提示"本月导出次数已用完"
- 导出文件过大: 分片打包，生成多个下载链接

**用户故事**: 作为用户，我想要管理我的隐私设置，以便控制个人信息的可见性和使用范围

**验收标准**:

**边界情况**:
- 检测到异常登录: 发送安全提醒通知
- 第三方解绑后无其他登录方式: 提示"请先绑定手机号或其他方式"

## 4. 非功能需求

### 4.1 性能要求

- **页面加载**: 首屏加载时间 < 2 秒（3G 网络环境下）
- **交互响应**: 用户操作响应时间 < 100ms
- **列表滑动**: 消息列表滑动帧率保持 60fps
- **图片加载**: 头像图片懒加载，支持渐进式加载
- **API 响应**: 普通接口响应时间 < 500ms，数据导出接口 < 3s（提交）

### 4.2 安全要求

- **数据传输**: 全站 HTTPS，TLS 1.3
- **数据存储**: 敏感数据（手机号、邮箱）加密存储（AES-256）
- **验证码**: 随机生成，不可预测，使用后立即失效
- **Token 机制**: JWT 双 Token（Access Token + Refresh Token）
- **防刷机制**: 接口请求频率限制（Rate Limiting）
- **隐私合规**: 符合 GDPR、个人信息保护法要求

### 4.3 兼容性要求

- **iOS**: 支持 iOS 13.0 及以上版本
- **Android**: 支持 Android 8.0 (API 26) 及以上版本
- **屏幕适配**: 支持 4.0 英寸至 6.7 英寸各类屏幕
- **暗黑模式**: 支持 iOS/Android 系统级暗黑模式切换
- **无障碍**: 支持 VoiceOver/TalkBack 屏幕阅读器

### 4.4 可用性要求

- **离线体验**: 支持离线查看已加载的个人资料
- **网络恢复**: 自动检测网络恢复，同步离线操作
- **错误处理**: 所有错误场景提供友好的用户提示
- **多语言**: 支持中文简体、中文繁体、英文（第一阶段仅中文简体）

### 5.3 数据模型

**用户表 (users)**
```sql
CREATE TABLE users (
    id BIGINT PRIMARY KEY,
    phone VARCHAR(11) UNIQUE NOT NULL,
    email VARCHAR(255),
    nickname VARCHAR(20),
    avatar_url VARCHAR(500),
    birthday DATE,
    gender SMALLINT, -- 0:未知 1:男 2:女
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    last_login_at TIMESTAMP
);
```

**验证码表 (verification_codes)**
```sql
CREATE TABLE verification_codes (
    id BIGINT PRIMARY KEY,
    phone VARCHAR(11) NOT NULL,
    code VARCHAR(6) NOT NULL,
    type VARCHAR(20), -- login, bind_phone, etc.
    expires_at TIMESTAMP NOT NULL,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    used_at TIMESTAMP
);
```

**消息表 (notifications)**
```sql
CREATE TABLE notifications (
    id BIGINT PRIMARY KEY,
    user_id BIGINT REFERENCES users(id),
    type VARCHAR(50) NOT NULL, -- system, activity, security
    title VARCHAR(100) NOT NULL,
    content TEXT,
    is_read BOOLEAN DEFAULT FALSE,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP
);
```

**导出记录表 (data_exports)**
```sql
CREATE TABLE data_exports (
    id BIGINT PRIMARY KEY,
    user_id BIGINT REFERENCES users(id),
    export_types JSONB, -- ['profile', 'messages', 'logs']
    format VARCHAR(10), -- json, csv
    status VARCHAR(20), -- pending, processing, completed, failed
    download_url VARCHAR(500),
    expires_at TIMESTAMP,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP,
    completed_at TIMESTAMP
);
```

## 6. UI/UX 设计

### 6.1 设计原则

- **移动优先**: 所有设计从移动端出发，再适配大屏
- **简洁清晰**: 减少视觉干扰，突出核心内容
- **即时反馈**: 操作后立即给出视觉或触觉反馈
- **容错设计**: 关键操作提供确认和撤销机制
- **一致性**: 遵循平台设计规范（iOS Human Interface / Material Design）

### 6.2 关键页面

| 页面 | 描述 | 主要元素 |
|------|------|---------|
| 登录页 | 手机号验证码登录入口 | Logo、手机号输入框、验证码输入框、获取验证码按钮、登录按钮 |
| 个人资料页 | 展示和编辑个人信息 | 头像、昵称、手机号、邮箱、生日、性别、编辑按钮 |
| 消息中心 | 消息列表和管理 | 分类标签、消息列表项（标题、摘要、时间、未读标记）、批量操作 |
| 设置页 | 应用设置入口 | 隐私设置、安全设置、通知设置、数据导出、关于我们、退出登录 |
| 数据导出页 | 导出配置和状态 | 数据类型选择、格式选择、邮箱输入、提交按钮、导出历史 |

### 6.3 交互流程

**登录流程**
```
启动 App
    ↓
检查登录状态 ──未登录──→ 显示登录页
    ↓                      ↓
已登录 ──→ 进入首页        输入手机号
                           ↓
                    点击获取验证码
                           ↓
                    输入验证码
                           ↓
                    点击登录 ──→ 登录成功 → 进入首页
                           ↓
                    登录失败 → 显示错误提示
```

**消息处理流程**
```
进入消息中心
    ↓
加载消息列表（分页）
    ↓
用户操作:
├── 点击消息 ──→ 标记已读 ──→ 跳转详情（如有）
├── 左滑消息 ──→ 显示删除/已读按钮
├── 下拉列表 ──→ 刷新获取最新
└── 点击筛选 ──→ 按类型过滤显示
```
