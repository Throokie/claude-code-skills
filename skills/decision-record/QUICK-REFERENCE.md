# Decision Record v2.0 - 快速参考卡片

> 需求演进与决策追踪器 - 把零散的用户原话，逼成可落地的方案

---

## 🎯 一句话理解

这不是简单的记录工具，而是一个**动态决策记录系统**：
- **输入**：零散的、看似愚蠢的用户原话
- **过程**：逐级剖析（1 级→2 级→N 级）+ 本地代码分析 + 多模型聚合
- **输出**：真正落地的【最终确定方案】

---

## 🚀 触发词

```
记录决策 / 需求演进 / 功能演进 / 决策追踪 / 需求推演 / 演进记录
```

## 📋 5 步工作流

```
┌─────────────────────────────────────────────────────────────┐
│  步骤 1：主题归纳                                            │
│  → 检查已有功能主题 → 是则追加，否则新建                      │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤 2：逐级意图剖析                                        │
│  → 1 级解读（表面层）→ 2 级解读（系统层）→ ⏸️ 暂停询问       │
│  "是否需要 3 级或更深解读？"                                 │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤 3：Claude Code 本地意见                                 │
│  → 读取相关代码 → 评估可行性 → 给出建议                      │
└─────────────────────────────────────────────────────────────┘
                            ↓
┌─────────────────────────────────────────────────────────────┐
│  步骤 4：多模型聚合搜索预留口                                │
│  → ⏸️ 暂停询问："是否需要外部多模型意见？"                    │

                            ↓

│  步骤 5：敲定最终方案                                        │
│  → ⏸️ 等待用户确认 → 记录【最终确定方案】                     │

```

## 💡 使用示例

### 示例：登录模块问题

```
你：需求演进 - 登录模块的验证码老是报错，而且我都不知道是没发出去还是填错了

AI:
## 步骤 1：主题归纳
→ 新建主题"登录模块用户体验改进"

## 步骤 2：逐级意图剖析
🕵️‍♂️ 1 级解读：
- 需求本质：用户需要明确的错误提示
🛠️ 1 级解决方案：
- 给验证码加错误文案提示

🏗️ 2 级解读：
- 需求本质：前后端状态协议缺失
⚙️ 2 级解决方案：
- 定义统一状态码协议

⏸️ 已完成 1 级和 2 级解读。是否需要继续进行更深维度的 3 级或 N 级解读？
- 回复"继续"进行更深剖析
- 回复"跳过"或"不需要"继续下一步

你：跳过

AI:
## 步骤 3：Claude Code 本地意见
→ 读取 auth_controller.ts, api_response.ts
→ 输出：建议抽象全局中间件

## 步骤 4：多模型聚合搜索预留口
⏸️ 请问您是否需要去询问其他大模型（如 GPT-4, Gemini 等）或者进行联网搜索，
以获取针对此问题的外部最佳实践？

你：需要，我去问问 GPT-4
（离开，返回后）
你：GPT-4 建议用 OAuth 2.0 标准 + Refresh Token

AI:
## 步骤 5：敲定最终方案
⏸️ 请确认最终方案：
- 回复"确定"或"就这么办"记录最终方案
- 回复"再想想"继续讨论

你：确定

AI:
✅ 最终确定方案记录完成
```

## 🎯 核心价值

| 场景 | 问题 | decision-record 解决方案 |
|------|------|-------------------------|
| **防甩锅** | "当初为什么要搞这么复杂？" | 翻开文档，看到是从"用户觉得字数不对"一步步被逼出来的 |
| **防遗忘** | "这个需求当时怎么定的？" | 完整记录从原始输入到最终方案的全过程 |
| **多模型整合** | " Claude Code 不了解业界最佳实践" | Claude Code 负责本地现实，外部模型负责仰望星空，人类架构师拍板 |
| **消除断层** | "立意和完成时不一样" | 只有【最终确定方案】里的内容才是真代码，保留思想转变轨迹 |

---

## ⚠️ 注意事项

1. **AI 不替代思考**：AI 负责推演、剖析、记录，人类负责决策
2. **暂停点必须等待**：步骤 2、4、5 都有暂停点，必须等待用户响应
3. **最终方案才是真理**：只有写在【最终确定方案】里的内容，才是真正要开发的
4. **状态追踪**：每个主题都有状态标记，方便追踪进度
